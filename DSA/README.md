## Exercise 2 - E-commerce Platform Search Function

### Objective

Implement Linear Search and Binary Search algorithms to search products in an e-commerce platform and compare their performance.

### Scenario

An e-commerce platform requires a fast and efficient search functionality to locate products based on their Product ID.

### Key Concepts

* Linear Search
* Binary Search
* Time Complexity Analysis
* Big O Notation
* Product Data Management

### Output

Linear Search Found: Watch

Binary Search Found: Watch

### Time Complexity

| Algorithm     | Best Case | Average Case | Worst Case |
| ------------- | --------- | ------------ | ---------- |
| Linear Search | O(1)      | O(n)         | O(n)       |
| Binary Search | O(1)      | O(log n)     | O(log n)   |

### Conclusion

Binary Search is more efficient for large datasets because it reduces the search space by half during each iteration. However, the data must be sorted before performing Binary Search.

## Exercise 7 - Financial Forecasting

### Objective

Implement a recursive algorithm to predict future financial values based on historical growth rates.

### Scenario

A financial forecasting application needs to estimate future investment values using annual growth rates.

### Key Concepts

* Recursion
* Mathematical Modeling
* Financial Forecasting
* Time Complexity Analysis

### Formula Used

Future Value = Present Value × (1 + Growth Rate)^Years

### Sample Input

* Present Value = 10000
* Growth Rate = 10%
* Years = 5

### Output

Future Value after 5 years = 16105.10

### Time Complexity

O(n)

The recursive function performs one recursive call for each year.

### Optimization

The recursive approach can be optimized using the direct mathematical formula, which avoids repeated recursive calls and improves performance.
